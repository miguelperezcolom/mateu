# Philosophy

Modern web development is overly complex.

We split applications into:

- backend
- frontend
- APIs

This creates duplication, synchronization issues, and unnecessary complexity.

Mateu takes a different approach.

## Define once

Your UI is defined once in your backend code.

## Generate, don't build

You don't manually build UIs.

They are generated from your code.

## Reduce moving parts

Fewer layers. Fewer bugs. Faster development.

---

Mateu is about reducing accidental complexity in software.
