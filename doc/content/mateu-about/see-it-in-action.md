# See it in action

The best way to understand Mateu is to try it.

The demos below all run against the **same backend**, deployed as two pods in a Kubernetes cluster on Hetzner infrastructure. Requests are load balanced between pods with no session affinity — a live demonstration of Mateu's stateless design.

Each demo uses a different frontend renderer and design system:

| Design system | Demo |
|---|---|
| Vaadin | [vaadin.mateu.io](https://vaadin.mateu.io/fluent/use-cases/rra) |
| Red Hat Design System | [redhat.mateu.io](https://redhat.mateu.io/fluent/use-cases/rra) |
| SAPUI5 | [sapui5.mateu.io](https://sapui5.mateu.io/fluent/use-cases/rra) |
| Oracle Redwood | [redwood.mateu.io](https://redwood.mateu.io/fluent/use-cases/rra) |
| Oracle Redwood (JET) | [redwood-oj.mateu.io](https://redwood-oj.mateu.io/fluent/use-cases/rra) |

All frontends are deployed as static sites on Netlify.
